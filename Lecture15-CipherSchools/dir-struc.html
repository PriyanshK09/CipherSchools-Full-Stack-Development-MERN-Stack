React App Folder Structure

Introduction

Understanding the folder structure of a React application is crucial for effective development and maintenance. When you create a new React app using Create React App, it comes with a well-organized folder structure. Let's explore the main components of this structure.

Folder Structure Overview


my-react-app/
├── node_modules/
├── public/
│   ├── favicon.ico
│   ├── index.html
│   ├── logo192.png
│   ├── logo512.png
│   ├── manifest.json
│   └── robots.txt
├── src/
│   ├── App.css
│   ├── App.js
│   ├── App.test.js
│   ├── index.css
│   ├── index.js
│   ├── logo.svg
│   ├── reportWebVitals.js
│   └── setupTests.js
├── .gitignore
├── package.json
├── README.md
└── yarn.lock / package-lock.json

Detailed Explanation

node_modules/

This folder contains all the project dependencies installed via npm. You typically don't need to worry about its contents, as it is managed by npm or yarn.

public/

The public directory contains the static assets of your application.

  • favicon.ico: The favicon of your app.
  • index.html: The main HTML file that is served when your app is requested.
  • logo192.png and logo512.png: Default logos used for various purposes.
  • manifest.json: A file used for configuring PWA (Progressive Web App) settings.
  • robots.txt: Instructions for search engine crawlers.

src/

The source directory contains the main source code of your application.

  • App.css: The CSS file for styling the App component.
  • App.js: The main App component.
  • App.test.js: The test file for the App component.
  • index.css: The CSS file for global styles.
  • index.js: The entry point of the React application. It renders the App component into the DOM.
  • logo.svg: The React logo.
  • reportWebVitals.js: A file for measuring the performance of your app.
  • setupTests.js: A file for setting up tests.

Root Files

  • .gitignore: Specifies which files and directories to ignore in Git.
  • package.json: Contains metadata about the project and lists dependencies.
  • README.md: Contains project documentation.
  • yarn.lock / package-lock.json: Ensures consistent installations across different environments.
