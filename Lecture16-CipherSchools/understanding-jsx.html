Understanding JSX

Introduction to JSX

JSX (JavaScript XML) is a syntax extension for JavaScript that is commonly used with React to describe what the UI should look like. JSX may look like HTML, but it's not exactly HTML. It allows you to write HTML-like code within JavaScript, making it easier to create and visualize React components.

JSX Syntax

Here is an example of a simple JSX element:

      
const element = <h1>Hello, world!</h1>;

This JSX code will be compiled to:

      
const element = React.createElement('h1', null, 'Hello, world!');

Embedding Expressions in JSX

You can embed any JavaScript expression in JSX by wrapping it in curly braces:

      
const name = 'Priyansh Khare';
const element = <h1>Hello, {name}!</h1>;

This allows you to dynamically render content based on your JavaScript variables.

JSX Attributes

You can use quotes to specify string literals as attributes, and curly braces to embed a JavaScript expression:

      
const element = <img src="https://example.com/logo.png" alt="Example Logo" />;
const element = <img src={user.avatarUrl} alt={user.name} />;

JSX Children

If a tag is empty, you can close it immediately with /> like XML:

      
const element = <img src="https://example.com/logo.png" alt="Example Logo" />;

JSX tags can contain children:

      
const element = (
  <div>
    <h1>Hello!</h1>
    <h2>Good to see you here.</h2>
  </div>
);

Conditional Rendering

You can embed any JavaScript expression in JSX by wrapping it in curly braces. This includes the conditional (ternary) operator:

      
const isLoggedIn = true;
const element = (
  <div>
    {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please sign in.</h1>}
  </div>
);
