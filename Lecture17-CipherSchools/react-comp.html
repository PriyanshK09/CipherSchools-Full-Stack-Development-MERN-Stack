Understanding React Components

Introduction to React Components

React components are the building blocks of a React application's UI. A component is a self-contained module that renders some output. We can write functional components using functions or class components using ES6 classes.

Class Components

Class components are ES6 classes that extend from React.Component. They must have a render() method which returns HTML (using JSX) to be displayed in the UI.

      
import { Component } from "react";

class App extends Component {
  name = "Shravan";

  render() {
    return (
      <>
        

          

            Hello from me {this.name}
          

          

            hellooo
          
 
        

        

          
This is a paragraph and the name is: {this.name}

        

      
    );
  }
}

export default App;

In the example above, we define a class component App that renders some JSX. The name property is used within the JSX to dynamically display content.

JSX in Class Components

JSX allows us to write HTML elements in JavaScript and place them in the DOM without using functions like createElement() or appendChild(). In the App component, we use JSX to create and style HTML elements:

      
<div>
  <h1 style={{ color: "red", textAlign: "center" }}>
    Hello from me {this.name}
  </h1>
  <p style={{ color: "blue", textAlign: "center" }}>
    hellooo
  </p> 
</div>
<div>
  <p>This is a paragraph and the name is: {this.name}</p>
</div>

JSX makes it easier to write and add HTML in React. Each component must return a single root element, which is why we use the React fragment syntax <>...</> to wrap multiple elements.
